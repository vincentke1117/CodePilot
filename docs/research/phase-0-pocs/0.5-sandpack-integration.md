## [外部事实] Sandpack 官方 API 引用

### A. `SandpackProvider` props 类型定义

来源：`github.com/codesandbox/sandpack/blob/main/sandpack-react/src/types.ts`（访问 2026-04-19）

```ts
export interface SandpackProviderProps<
  Files extends SandpackFiles = SandpackFiles,
  TemplateName extends SandpackPredefinedTemplate = SandpackPredefinedTemplate
> extends SandpackRootProps<Files, TemplateName>, React.HTMLAttributes<HTMLDivElement> {
  options?: SandpackInternalOptions<Files, TemplateName>;
  children?: React.ReactNode;
}

export interface SandpackOptions {
  bundlerURL?: string;            // ← 这里！不在 customSetup
  externalResources?: string[];
  activeFile?: string;
  visibleFiles?: string[];
  initMode?: SandpackInitMode;
  autorun?: boolean;
  recompileMode?: "immediate" | "delayed";
  recompileDelay?: number;
  logLevel?: SandpackLogLevel;
  startRoute?: string;
  skipEval?: boolean;
  fileResolver?: FileResolver;
  classes?: Record<string, string>;
  // ...UI flags (showTabs/showConsole/showNavigator/...)
}

export interface SandpackSetup {
  dependencies?: Record<string, string>;
  devDependencies?: Record<string, string>;
  entry?: string;
  environment?: SandboxEnvironment;
  npmRegistries?: NpmRegistry[];
  exportOptions?: SandpackExportOptions;
}

export type SandpackFiles = Record<string, string | SandpackFile>;

  code: string;
  hidden?: boolean;
  active?: boolean;
  readOnly?: boolean;

```

**Codex 原判断复核：** `bundlerURL` 位于 `options.bundlerURL`，**不在** `customSetup`。确认。

### B. `SandpackPreview` props + 转发 ref

来源：`github.com/codesandbox/sandpack/blob/main/sandpack-react/src/components/Preview/index.tsx`

```ts
export interface PreviewProps {
  style?: React.CSSProperties;
  showNavigator?: boolean;
  showOpenInCodeSandbox?: boolean;
  showRefreshButton?: boolean;
  showRestartButton?: boolean;
  showSandpackErrorOverlay?: boolean;
  showOpenNewtab?: boolean;
  actionsChildren?: JSX.Element;
  children?: JSX.Element;
  startRoute?: string;

export interface SandpackPreviewRef {
  getClient: () => InstanceType<typeof SandpackClient> | null;

// React.forwardRef<SandpackPreviewRef, PreviewProps & React.HTMLAttributes<HTMLDivElement>>
```

**关键结论：** `SandpackPreview` **没有** `sandbox` prop，转发 ref 只暴露 `getClient()` + `clientId`，**不暴露 iframe DOM**。

### C. 低层 `SandpackRuntime` 构造器对 iframe 的处理

来源：`github.com/codesandbox/sandpack/blob/main/sandpack-client/src/clients/runtime/index.ts`

- 构造签名：`SandpackRuntime(iframe, content, options)`；`iframe` 可为 `HTMLIFrameElement` 或 CSS selector string
- 用户传 `HTMLIFrameElement` 时：`this.element = selector; this.iframe = selector;` — **直接复用用户 iframe**
- sandbox 属性处理：`if (!this.iframe.getAttribute("sandbox")) { this.iframe.setAttribute("sandbox", "allow-forms allow-modals allow-popups allow-presentation allow-same-origin allow-scripts allow-downloads allow-pointer-lock"); }`
- **即用户预先在 iframe 上 `setAttribute("sandbox", "...")`，Sandpack 不会覆盖。**

### D. `ClientOptions.externalResources`

来源：`github.com/codesandbox/sandpack/blob/main/sandpack-client/src/types.ts`

```ts
externalResources?: string[]; // "Paths to external resources"
```

Sandpack bundler 会把这些 URL 注入 preview 文档（`.css` → `<link>`、`.js` → `<script>`）。适合挂 Tailwind CDN。

### E. Electron `session.webRequest.onHeadersReceived`

来源：`electronjs.org/docs/latest/api/web-request`

```ts
session.defaultSession.webRequest.onHeadersReceived(
  { urls: ["https://sandpack-bundler.codesandbox.io/*"] },
  (details, callback) => {
    const headers = { ...details.responseHeaders };
    headers["Content-Security-Policy"] = [
      "default-src 'self' https://*.codesandbox.io; script-src 'self' 'unsafe-eval' https://*.codesandbox.io https://cdn.jsdelivr.net; style-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net; connect-src 'self' https://*.codesandbox.io https://registry.npmjs.org;",
    ];
    callback({ responseHeaders: headers });
  }
);
```

### F. Sandpack 自托管 bundler

来源：`github.com/codesandbox/sandpack-bundler` README（访问 2026-04-19）

- `yarn dev` → 端口 **1234**；`yarn build && yarn start` → 端口 **4587**

## [iframe 安全控制路径] s1 / s2 / s3 / s4 可行性论证

| 路径 | 机制 | 可行性 | 证据 | 复杂度 |
|---|---|---|---|---|
| **s1** Electron CSP 注入 | `onHeadersReceived` 为 bundler origin 重写 CSP | **可行** | Electron 官方 API ([E])；但只能覆盖**我们可拦截的 origin**（默认 `sandpack-bundler.codesandbox.io`），不能改 iframe 内部 `sandbox` 属性（sandbox 是父页面设置，CSP 不替代） | 中 |
| **s2** 低层 `sandpack-client` + 自建 iframe | 手动 `createElement('iframe') → setAttribute('sandbox', '...')` → `new SandpackRuntime(iframe, content, options)` | **完全可行（推荐）** | 源码明确显示 ([C])：用户 iframe 预设 sandbox 时不被覆盖 | 高（放弃 `@codesandbox/sandpack-react` 的 ContextProvider） |
| **s3** `SandpackPreview` + ref 运行时改 sandbox | 通过 `SandpackPreviewRef.getClient().iframe` 拿到 iframe 后 `setAttribute` | **不可行 / 脆弱** | `SandpackPreviewRef` 只暴露 `getClient() + clientId`，**不暴露 iframe** ([B])；通过 `getClient().iframe` 属于 client 内部实现非公共 API；且 sandbox 属性改后浏览器不会重新应用（需 iframe 重载），Sandpack dispatch refresh 会走相同 iframe，sandbox 生效时机不确定 | 低（但不稳定） |
| **s4** 接受默认策略 | 不干预，记录默认行为 | **可行但边界不足** | 默认 sandbox = `allow-forms allow-modals allow-popups allow-presentation allow-same-origin allow-scripts allow-downloads allow-pointer-lock` ([C])；**包含 `allow-same-origin`** — 意味着 iframe 页面能访问其同源资源（bundler.codesandbox.io），这在跨域角度仍隔离父页面，但 Phase 2.5 想删 `allow-same-origin` 就做不到 | 低 |

**推荐策略（分档）：**

1. **Phase 2.1 默认落 s4**（接受默认），Phase 2.5 评估 4 个安全样本后决定是否升级
2. **若 2.5 要求删 `allow-same-origin` 或强化 CSP** → 升级到 **s2**（低层 client + 自建 iframe）；同时叠加 **s1**（自托管 bundler + `onHeadersReceived` 注入 CSP）
3. **s3 永远不采用** — 不稳定且依赖非公共 API
4. **s1 单独使用不够**（管不到 iframe 的 `sandbox` attribute）；但作为 s2 的补充很好

## [设计] `SandpackPreview` 组件草案

假设已装 `@codesandbox/sandpack-react@^2.20`（仅 Phase 2.1 开工时执行）。

### `src/components/editor/SandpackPreview.tsx`

```tsx
"use client";

import { useMemo } from "react";
import dynamic from "next/dynamic";
import type { SandpackFiles, SandpackSetup } from "@codesandbox/sandpack-react";
import { ErrorBoundary } from "@/components/common/ErrorBoundary";
import { Skeleton } from "@/components/ui/skeleton";

const SandpackProvider = dynamic(
  () => import("@codesandbox/sandpack-react").then((m) => m.SandpackProvider),
  { ssr: false, loading: () => <PreviewSkeleton /> },
);
const SandpackLayout = dynamic(
  () => import("@codesandbox/sandpack-react").then((m) => m.SandpackLayout),
  { ssr: false },
);
const SandpackPreviewInner = dynamic(
  () => import("@codesandbox/sandpack-react").then((m) => m.SandpackPreview),
  { ssr: false },
);

// Phase 2.1 白名单（用户反馈可增补）
const ALLOWED_DEPS: Record<string, string> = {
  "react": "^18.3.1",
  "react-dom": "^18.3.1",
  "lucide-react": "^0.468.0",
};

// Tailwind v4 通过 externalResources 注入（Phase 2.1 初版最省事）
const EXTERNAL_RESOURCES = [
  "https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4",
];

export interface SandpackPreviewProps {
  /** 文件路径（决定挂载点，如 "App.tsx"）；缺省时默认 /App.tsx */
  filePath?: string;
  /** 源码内容 */
  content?: string;
  /** 离线场景下本地自托管 bundler URL，如 http://localhost:1234 */
  bundlerURL?: string;
}

function inferMountPath(filePath?: string): string {
  if (!filePath) return "/App.tsx";
  const base = filePath.split("/").pop() ?? "App.tsx";
  return base.endsWith(".tsx") || base.endsWith(".jsx")
    ? `/${base}`
    : "/App.tsx";
}

export function SandpackPreview(props: SandpackPreviewProps) {
  const { filePath, content, bundlerURL } = props;

  const { files, setup, activeFile } = useMemo(() => {
    const mount = inferMountPath(filePath);
    const files: SandpackFiles = {
      [mount]: { code: content ?? "export default () => null;", active: true },
      // Phase 2.1 Part 2：注入 shadcn 子集以支持 @/components/ui/* 别名（C 类 fixture）
      // "/components/ui/button.tsx": { code: SHADCN_BUTTON_SRC, hidden: true },
    };
    const setup: SandpackSetup = { dependencies: ALLOWED_DEPS };
    return { files, setup, activeFile: mount };
  }, [filePath, content]);

  return (
    <ErrorBoundary fallback={<PreviewError />}>
      <SandpackProvider
        template="react-ts"
        files={files}
        customSetup={setup}
        options={{
          activeFile,
          bundlerURL,                     // 离线 → 本地 1234；在线 → undefined 走官方
          externalResources: EXTERNAL_RESOURCES,
          recompileMode: "delayed",
          recompileDelay: 400,
          autorun: true,
        }}
      >
        <SandpackLayout>
          <SandpackPreviewInner
            showOpenInCodeSandbox={false}
            showRefreshButton
            showSandpackErrorOverlay
            style={{ height: "100%", minHeight: 480 }}
          />
        </SandpackLayout>
      </SandpackProvider>
    </ErrorBoundary>
  );
}

function PreviewSkeleton() {
  return <Skeleton className="h-full min-h-[480px] w-full" />;
}
function PreviewError() {
  // i18n：`filePreview.sandpackError`
  return (
    <div className="flex h-full items-center justify-center text-sm text-muted-foreground">
      预览组件加载失败，请刷新重试
    </div>
  );
}
```

## [A/B/C fixture 预期] 行为表

| Fixture | 代码特征 | 主路径（react-ts template + ALLOWED_DEPS） | 预期 |
|---|---|---|---|
| **A** 无 import JSX snippet | `export default () => <h1 className="text-3xl">Hi</h1>` | Sandpack 把 snippet 包进 `/App.tsx`，Tailwind 通过 `externalResources` 生效 | ✅ 100% 成功 |
| **B** 真实 AI 输出 | `import React, { useState } from 'react'; import { Heart } from 'lucide-react'; export default function Card(){...}` | `customSetup.dependencies` 白名单覆盖 `react` / `react-dom` / `lucide-react` | ✅ 成功；不在白名单的依赖 → bundler 报 404，ErrorOverlay 提示 |
| **C** 项目别名 | `import { Button } from '@/components/ui/button'` | 默认 **失败**（bundler 不认 Next.js tsconfig paths）；**缓解方案**：`files` 注入 `/components/ui/button.tsx` 虚拟文件（shadcn 子集 hidden:true），并在 Sandpack 沙箱 `tsconfig.json` 的 `paths` 里映射 `@/*` | ⚠️ 部分可行；Phase 2.1 P1 先给"别名不支持"提示条（i18n `filePreview.aliasNotSupported`），Phase 2.1 P2 再补虚拟 shadcn |

C 类当前最小可行策略 = **注入一个 `/tsconfig.json` 虚拟文件**：

```json
{ "compilerOptions": { "baseUrl": ".", "paths": { "@/*": ["./*"] } } }
```

+ `/components/ui/button.tsx` hidden。Sandpack bundler 支持 tsconfig paths 解析（源自 `sandpack-bundler` 的 `tsconfig-paths-webpack-plugin` 集成）。

## [离线 bundler] 自托管步骤

### macOS

```bash
cd ~/dev
git clone https://github.com/codesandbox/sandpack-bundler.git
cd sandpack-bundler
yarn           # 若无 yarn: brew install yarn
yarn dev       # http://localhost:1234
# 或生产模式：yarn build && yarn start → http://localhost:4587
```

### Windows

```powershell
cd $HOME\dev
git clone https://github.com/codesandbox/sandpack-bundler.git
cd sandpack-bundler
corepack enable    # Node 16.10+ 自带 corepack，启用 yarn
yarn
yarn dev          # http://localhost:1234
```

### 在 `SandpackProvider` 引用

```tsx
<SandpackProvider options={{ bundlerURL: "http://localhost:1234" }} ... />
```

**CodePilot 接入注意：**

- 本地 bundler 不随 app 打包（体积过大，依赖 Node 运行时）；仅作为"离线开发者手动启动"的 fallback
- Phase 2.1 默认用官方 CDN `https://sandpack-bundler.codesandbox.io`
- 用户偏好项 `preview.sandpackBundlerURL`（string / null）暴露在 Settings，默认空走官方

## [验收条件] Phase 2.1 落地 smoke test

1. **主路径在线** — 官方 CDN 能通；A/B fixture 渲染成功；C fixture 出现"别名不支持"提示条或成功渲染（取决于 P1/P2）
2. **bundler 配置验证** — 把 `options.bundlerURL` 指向 `http://localhost:1234`（本地 clone 的 bundler），A/B 仍能渲染
3. **iframe 默认 sandbox 断言** — CDP `$("iframe").getAttribute("sandbox")` 输出包含 `allow-scripts allow-same-origin`（与 [C] 的默认字符串一致）→ 确认 s4 默认生效
4. **代码切换热更新** — `content` prop 变更 → `recompileMode: "delayed"` + `recompileDelay: 400` 验证 400ms 后预览更新
5. **动态加载 bundle 尺寸** — `npm run build` 后 `Route (app) /chat/[id]` 的 First Load JS 不包含 `@codesandbox/sandpack-react`（验证 `dynamic` 按需加载生效）
6. **错误边界** — 故意写 `throw new Error("x")` 组件 → `SandpackErrorOverlay` 在 iframe 内展示；外层 ErrorBoundary 不被触发
7. **Tailwind 生效** — A fixture 的 `text-3xl` 视觉上呈现 3xl 字号（截图比对）
8. **安全路径决策日志** — Phase 2.5 验收 4 个 iframe 安全样本后，决策日志注明"s4 接受默认 / 升级到 s2"中的一项

## 结论与下一步

- Sandpack 官方 API 足以覆盖 A/B fixture；C fixture 可通过 `files` + `tsconfig.json` 虚拟文件解决
- **推荐 Phase 2.1 初版走 s4**（接受默认 sandbox）+ `@codesandbox/sandpack-react` 高层 API；保留 s2 升级路径
- 离线 bundler 走源码启动，**不可 `npm i`**；Phase 2.1 只做配置开关，不内嵌 bundler
- Tailwind v4 通过 `options.externalResources` 挂 CDN；Phase 2.1 初版够用，Phase 2.5 若需离线走 `customSetup.files` 内联静态 CSS

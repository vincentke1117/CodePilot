# CodePilot 0.55.0-preview.5 内测说明

> 面向小范围内测用户。此版本不是正式 Release，不走自动更新，不代表最终 UI 与能力形态。

## 本次内测目标

这轮主要验证大重构后的基础可用性：应用能不能稳定启动、已有数据能不能继续使用、不同执行引擎能不能正常发消息、设置页是否不卡死，以及 macOS 包的安装与图标表现是否正常。

请优先反馈 **P0 / P1** 问题。视觉细节、文案偏好、轻微排版问题可以记录，但不作为本轮是否可继续测试的主要判断。

## 安装前注意

本预览包会读取现有 CodePilot 数据，不会使用独立测试数据目录。安装前建议先备份：

- `~/.codepilot/codepilot.db`
- `~/Library/Application Support/CodePilot`（如果存在）

安装和测试时请注意：

- 先完全退出正在运行的 CodePilot，再安装预览版。
- 不要同时运行稳定版和预览版。
- 此包为内部测试包，未公证；macOS 首次打开可能需要右键打开，或在 System Settings → Privacy & Security 中允许打开。
- 当前包只面向 Apple Silicon Mac。Intel Mac 和 Windows 包会单独验证。

## 这轮主要更新

### 执行引擎与能力接入

- 重构了 Runtime / Harness 架构，让 CodePilot 内置能力更清楚地区分「可调用」「可感知但不可调用」「不支持」。
- CodePilot Native Runtime 补齐了 Memory、Widget、Tasks/Notify、Image、Media、Assistant Buddy 等基础能力。
- Codex Runtime 接入了更多 CodePilot 原生能力，包括 Memory、Widget、Tasks/Notify、Dashboard、CLI Tools；其中会产生副作用的操作会走审批。
- Settings → Runtime 里新增能力清单，用用户能理解的方式说明当前引擎支持什么、哪些能力需要确认、哪些能力需要切换引擎。

### 启动与模型加载修复

- 修复旧 Codex CLI 抢先于 Codex.app 被发现导致的 Codex 启动失败。
- 修复 Codex app-server 启动失败时页面长时间卡在加载状态的问题。
- 修复 Claude Code 下裸 `sonnet` 别名导致模型不可用的问题。
- 修复 Settings 概览、执行引擎、模型列表不应被 Codex 慢启动整体拖住的问题。

### macOS 体验

- macOS 视觉改为浮动卡片布局，左侧列表、主聊天区、右侧栏和文件树使用统一卡片框架。
- 修复 macOS 菜单栏 / 顶部托盘图标在打包后不可见或不清晰的问题。
- 图标系统迁移到统一的 CodePilotIcon 入口，减少重复图标和表意不清。

### 上下文统计

- 新增上下文用量的分类展示，用于区分对话、规则、技能、工具调用等来源。
- 对暂时没有真实数据来源的类别会隐藏，不再显示误导性的 0 或估算值。

### 打包与稳定性

- 打包流程会重新编译 native 依赖，避免 `better-sqlite3` ABI 不匹配导致启动闪退。
- 测试隔离与 pre-commit 稳定性做了修复，减少本地测试污染真实数据库的风险。

## 建议测试路径

请尽量按顺序测试，并记录每一步是否通过。

1. 安装并启动预览版。
2. 确认版本显示为 `0.55.0-preview.5`。
3. 确认旧会话、Provider、模型、素材库还能看到。
4. 打开 Settings → Overview，确认不会长时间卡在加载中。
5. 打开 Settings → Runtime，确认能力清单能正常显示。
6. 新建聊天，用默认执行引擎发送一条普通消息。
7. 切换到 Claude Code，发送一条普通消息。
8. 如果使用 Codex，切换到 Codex Runtime，确认 Codex 状态不会长时间卡死。
9. 如果有 Codex Account，测试 Memory 或 Widget 这类只读能力。
10. 测试一个需要审批的能力，例如创建任务、发送通知、Dashboard 写入或 CLI 写操作，确认会弹出确认。
11. 创建一个 3 分钟定时提醒，确认至少 app 内 toast 或系统通知有一种可见。
12. 退出并重新打开 CodePilot，确认状态不会丢失。

## 问题级别

| 级别 | 定义 | 示例 |
|------|------|------|
| P0 | 无法继续测试，或可能造成数据损坏 | 无法安装、无法启动、启动闪退、已有会话丢失、数据库损坏、无法发送任何消息 |
| P1 | 核心路径不可用，但可以继续使用其他功能 | 某个 Runtime 完全不可用、模型列表一直加载、Settings 卡死、Codex/Claude Code 无法发消息 |
| P2 | 可绕过但影响体验 | UI 错位、文案不清、图标表意不准、通知不符合预期但有其他提示 |
| P3 | 建议或偏好 | 视觉风格建议、交互优化建议、希望新增的能力 |

本轮最关注 P0 / P1。

## 已知边界

- 这是内部测试包，不是正式发布版本。
- Windows 包还需要独立构建和真机验证。
- macOS 系统通知可能受系统权限、专注模式、前台应用抑制影响；如果 app 内 toast 出现但系统横幅不出现，也请记录。
- Codex 是否主动调用某些能力会受模型行为影响；请记录实际提示词和结果，不要只写「它没调用」。
- 如果遇到启动或 Runtime 问题，请附带日志，尤其是 Codex / Claude Code 相关错误。

## 一句话目标

v0.56.x 不追新能力，专门修“可信主路径”：对话不中断、上下文不串、文件不误改、预览不被多余确认阻断、安装更新可靠、出错可诊断、外部协作有质量门禁。

## 用户能看到什么

- Stop / 中断 / resume 后，同一个会话能继续发送，不再需要重启应用或新建会话。
- 长对话与上下文压缩失败时有明确原因和可诊断信息，不再默默串上下文或生成空 assistant 消息。
- 上下文用量不再显示假百分比或超过 100%；输入框和浮层只展示有真实来源的已用量与构成比例。
- 图片 / 文件等普通上下文附件不再触发“上下文太多，确认发送”这类阻断式横幅；Codex Runtime 下，图片 + 文本混合输入能按 Enter 正常发送，发送成功后输入框不会残留旧内容。
- 外部 Markdown / HTML / 本地网页预览直接打开，不再弹出多余“确定”确认；本地 HTML / 网页预览支持页面自身 JS 驱动的动态样式与交互，不只渲染静态 HTML。
- `@file` / 附件 / 可编辑源文件的语义清楚，AI 写操作不会误改 `.codepilot-uploads` 副本。
- Windows 安装、更新提示、日志导出与崩溃定位更可靠。
- GitHub issue / PR 有明确优先级、模板和 CI 门禁，外部贡献不再把主线拉散。

## 不做什么

- 不在 v0.56.x 主线新增 provider、bridge、Media Studio 能力、模块市场能力或大 UI 改版。
- 不把 Notion 文档提到的 issue 根因直接当事实；所有 GitHub issue 先重新核验当前状态、复现步骤和本地代码证据。
- 不把安全大项混入小修：API key 加密落盘、文件搜索索引、Runtime Capability Adapter 若进入 v0.56.x，必须单独拆 Phase / 执行计划。
- 不由 Codex 修改产品代码。产品 / 运行时 / Electron / 样式 / 构建脚本改动仍由 Claude Code 实施。

## Phase 2 — Session / Context / Stream / Composer 可信主链路

**用户能看到什么**：对话不再“半停半活”；resume 失败、context overflow、provider error 都有明确原因，坏 session 不继续污染下一轮；上下文用量不再用不可靠百分比吓人；图片等普通附件不会再弹出阻断式“确认发送”横幅；Codex 多模态输入能正常发送，发送后草稿清空。

**不做什么**：不重写 Runtime 架构；不把上游 Codex core 问题冒充 CodePilot 已修。

**怎么验收**：

- Codex Runtime 下：长 turn → Stop → 同会话下一条消息能被接受。
- Resume 400 / 失败不会产生空 assistant message，也不会继续沿用坏 SDK session id。
- Context source breakdown 能解释当前 thread / memory / attachments / summary / tool result 来源。
- 上下文总长度无法可靠取得时，任何 UI 都不显示百分比、剩余上下文或“未使用”数值；输入框右下角只显示当前已用上下文数值；浮层只显示已用上下文数值与不同上下文种类之间的占比。
- 图片 / 文件等用户主动添加的普通上下文，只能出现非阻断提示或用量说明，不能要求用户再点一次“确认发送”。阻断式确认只保留给真实危险动作：写文件、删文件、执行命令、提权，或把外部内容发送给模型 / 跨越权限边界的数据访问；单纯 Markdown / HTML / 网页预览不得二次确认阻断。
- 压缩失败时可回滚，不污染下一轮。
- Codex Runtime 下，文本 + 图片混合内容按 Enter 能发送；发送成功后 composer draft / pending attachments / local echo 状态一致清空，失败时才保留可重试草稿。
